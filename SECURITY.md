# Security Policy

## Reporting a Vulnerability

If you discover a security issue in BRIDGE, please do not open a public issue
with exploit details.

Instead, report it privately to the project maintainers through a private
channel available to the repository owners.

## Scope

Security concerns include, for example:
- credential or token exposure
- unintended disclosure of private infrastructure details
- unsafe handling of paths, files, or runtime configuration
- package or CLI behaviors that create avoidable security risks
